Key Engineering Decisions

→
Client-side date filtering — avoids Firestore composite index requirements; all date logic runs in JavaScript
→
Base64 document IDs — habit log compound keys (uid+habitId+date) encoded with btoa() to prevent Firestore path corruption
→
Optimistic UI with toggle guards — in-flight state per habit+date prevents race conditions on double-tap
→
Zero-cost architecture — Firebase Spark + Vercel free tier; $0/month, sustainable indefinitely
